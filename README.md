# kakao 企画書（v0.1）

## 1. 目的

* ユーザーが「今の自分の覚醒状態」と「睡眠への残響」を直感で把握し、**飲み過ぎ・遅い時間の摂取**を減らす。
* 数字だけの健康アプリに寄せず、**時間経過＝キャラ変化**で行動を動かす。

## 2. 課題
- 現状：眠くなったら とりあえずカフェインを入れる。私生活に影響が出たら改善する。という繰り返し。
- 理想：私生活に影響が出る前に日々対策する。適量のカフェインで生活/健康に影響が出ないようにする。
- 課題：カフェインの飲みすぎ⇨生活に影響が出てしまう。"今”を優先してカフェインを入れる。少し眠いなどの感覚に頼ってカフェインを入れる
- 作戦：カフェインの持続時間を可視化すれば、必要以上にカフェインを入れることがなくなる。



## 3. ターゲット

* コーヒー/エナドリを日常的に飲む学生・エンジニア・デスクワーカー
* 「集中したい」「でも睡眠は壊したくない」層

## 4. 提供価値（コア）

* 摂取入力 → 残量推定 → **キャラの状態が時間で変化**（集中/冴え/ソワソワ/落ちる）
* “残量の見える化”を、毎回開きたくなる形（キャラ）に変換する。

## 5. プロダクトコンセプト

* **kakao = カフェインの残響を飼うアプリ**
* 数値の正確性を過剰に追わず、**意思決定（今飲む？やめる？）を助ける**方向に振る。

## 6. ユーザーストーリー

1. 朝：コーヒー1杯を記録 → キャラが「起動」
2. 午後：残量がまだ高い → 追いカフェインを抑制
3. 夕方：睡眠予定時刻が近い → “夜に残る”警告＋キャラが不穏化
4. 翌日：ログが残って「昨日の失敗」が見える →学習が起きる

## 7. 主要機能（MVP→拡張）

### MVP

* 摂取記録：プリセット（コーヒー/エナドリ）＋mg手入力
* 残量タイムライン：現在の残量、0になるまでの目安
* キャラ3D表示：状態3〜5段階で見た目/アニメ変化
* 通知：就寝X時間前に「まだ残る」警告（任意）

### 拡張

* 生活リズム設定：就寝/起床、カフェイン制限時間の自動提案
* Widget / Live Activity（iOS強みを活かす）
* 個人最適化：半減期や感受性をユーザーが調整（“体感に合わせる”）
* キャラ育成要素：連続記録・夜カフェ回避で成長（ただしやり過ぎると胡散臭くなる）

## 8. “カフェイン残量”のモデル

* 初期は **単純な半減期モデル**で十分（精密医療をやる気なら負ける）

  * 残量 = Σ(摂取量 × 0.5^(経過時間 / 半減期))
* 半減期はデフォルト固定（例：5時間）＋将来は調整可能にする
* 注意書き：推定であり、個人差が大きい（ここを誤魔化すと信用が落ちる）

## 9. 3D体験設計（検討）

* 勝ち筋：**状態変化が一目で分かる**（表情/姿勢/呼吸/色味/モーション）
* 実装の現実：

  * 3Dの難所はコードではなく **モデル制作（リグ/表情/アニメ/最適化）**
  * MVPは「1体＋アニメ切替」より、**状態別に3Dを差し替え**た方が事故が少ない

## 10. 技術

* SwiftUI（UI）
* SceneKit（3D表示：SceneViewで載せる）
* ローカル保存：UserDefaults → 伸びたらSwiftData
* 通知：UNUserNotificationCenter
* 最初からクラウド同期は不要（逃げ）

## 11. 画面案（最小）

* Home：キャラ＋現在状態（ゲージ/一言）
* Timeline：今後12時間の残量カーブ
* Log：摂取履歴（編集・削除）
* Settings：就寝時刻、半減期、通知ON/OFF、プリセット編集

## 12. 差別化
* 既存：数字の記録アプリ
* kakao：**数字 → 感情的に分かる視覚表現（キャラ）**
* “可愛い”ではなく **判断が速くなるUI** を作れ。可愛さだけなら即死。

## 13. KPI

* 1日あたり摂取記録回数（入力継続）
* 就寝X時間前の摂取率（減っているか）
* 7日継続率（キャラが機能してるか）
